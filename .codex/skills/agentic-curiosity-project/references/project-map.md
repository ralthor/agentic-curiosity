## Runtime And Tooling

- `README.md`: Canonical high-level repo overview, API examples, and current Docker or deployment workflow.
- `pyproject.toml`: `uv`-managed Python 3.11+ project. Current direct dependencies are Django, Gunicorn, and `openai`.
- `uv.lock`: Lockfile generated by `uv`.
- `manage.py`: Django management entry point.
- `Dockerfile`: Container build for the app image. Installs `uv`, syncs the locked production dependencies, copies the repo, and starts `docker-entrypoint.sh`.
- `docker-entrypoint.sh`: Container startup script. Runs migrations, optionally runs `runserver`, otherwise collects static files and starts Gunicorn.
- `compose.yml`: Local Docker stack with `web`, `redis`, and `nginx`. Mounts `./docker-data` into the web container as `/data`, mounts `./redis-data`, and publishes Nginx on `${NGINX_HTTP_PORT:-80}`.
- `nginx.conf`: Local reverse-proxy and static-file config for the Compose stack.
- `deploy/bootstrap.sh`: Server-side bootstrap helper that writes `compose.yml` and `nginx.conf` next to itself from `.env`, creates persistent data directories, validates the Compose file, and optionally starts the stack.
- `db.sqlite3`: Current local development database.

## Django Project Package

- `agentic_curiosity/settings.py`: Global Django settings. Loads `.env`, resolves the SQLite path from `DJANGO_DB_PATH`, includes `core`, `ai_chat`, and `chat_api`, configures OpenAI and AI chat settings, exposes login rate-limit settings, proxy and TLS settings, static root settings, and console logging for the `ai_chat` logger.
- `agentic_curiosity/env.py`: Minimal dotenv-style parser plus `env_bool()`, `env_list()`, and `env_int()` helpers used by settings.
- `agentic_curiosity/urls.py`: Root URL config. Routes `""` to `core.urls`, `/api/chat/` to `chat_api.urls`, and `/admin/` to Django admin.
- `agentic_curiosity/tests.py`: Tests env-file loading and SQLite path resolution behavior.
- `agentic_curiosity/asgi.py` and `agentic_curiosity/wsgi.py`: Standard Django entry points.

## Current Apps

### `core`

- `core/views.py`: Renders the browser chat console at `/` and the course-studio page at `/course-topics/`.
- `core/urls.py`: Wires the root route and the course-studio page.
- `core/templates/core/home.html`: Browser chat console for login, course selection, session creation or retrieval, active-question display, answer submission, hint requests, skip, and full-answer requests. Stores browser state in `localStorage` and renders Markdown responses client-side.
- `core/templates/core/course_topics.html`: Browser UI for creating courses, listing stored course content, and importing questions into an existing course.
- `core/tests.py`: Route and template smoke tests for the browser pages.

### `ai_chat`

- `ai_chat/__init__.py`: Public exports for the app.
- `ai_chat/agents.py`: Provider-agnostic `Agent` base class. Shared logic lives here.
- `ai_chat/openai_agent.py`: `OpenAIAgent` implementation using the OpenAI SDK chat completions client and settings-backed API key, organization, project, and base URL resolution.
- `ai_chat/models.py`: Persisted tutoring/session tables: `ChatSession`, `QuestionPresentation`, `QuestionAttempt`, and `LearnerQuestionState`.
- `ai_chat/views.py`: Intentionally empty. This app exposes Python utilities and models, not HTTP views.
- `ai_chat/management/commands/compact_chat_contexts.py`: Deprecated command that raises a `CommandError` because the question-first engine no longer uses global chat-context compaction.
- `ai_chat/admin.py`: Django admin registration for sessions, presentations, attempts, and learner state.
- `ai_chat/exceptions.py`: `AgentConfigurationError` and `AgentResponseError`.
- `ai_chat/tests.py`: Behavioral tests for the base class and `OpenAIAgent`.

### `chat_api`

- `chat_api/models.py`: `Course`, `CourseTopic`, `QuestionType`, `CourseQuestion`, `ApiToken`, and `LoginRateLimit`.
- `chat_api/auth.py`: Parses `Authorization: Token ...` or `Authorization: Bearer ...` and resolves the authenticated Django user from `ApiToken`.
- `chat_api/rate_limits.py`: Sliding-window login throttling keyed by normalized username plus a client identifier derived from `X-Real-IP`, `X-Forwarded-For`, or `REMOTE_ADDR`.
- `chat_api/question_selector.py`: Selects the next question using explicit question overrides, topic overrides, the `same_topic` strategy override, unseen or due or fallback pools, and topic-balancing heuristics.
- `chat_api/progress.py`: Computes course/topic coverage and mastery summaries and serializes the active question.
- `chat_api/services.py`: Core question-first tutoring workflow: session creation, interaction classification, hint or mark or full-answer prompt construction, learner-state updates, question advancement, and AI-generated example-answer caching.
- `chat_api/views.py`: Token-authenticated JSON endpoints for login, token issuance, course listing or creation, question import, session creation, session lookup, and chat replies.
- `chat_api/urls.py`: Routes under `/api/chat/`.
- `chat_api/admin.py`: Django admin registration for courses, topics, question types, questions, and API tokens.
- `chat_api/tests.py`: Model, selector, progress, login throttle, course import, session, and interaction tests for the authenticated chat flow.

## Runtime Invariants

- The active runtime is question-first. The older full-session planner or chat-context compaction flow is not the current engine.
- `Course` is the root content object. `CourseTopic` groups questions for coverage and mastery reporting. `QuestionType` owns reusable `hint_prompt` and `mark_prompt` text. `CourseQuestion` stores authored question content plus optional `sample_answer`, `example_answer`, and `marking_notes`.
- `ChatSession` owns the selected course, optional selector overrides, the optional `same_topic` strategy override, and the `active_presentation`. Later interactions should follow that session state rather than silently switching courses or questions.
- `QuestionPresentation` tracks the currently served question and how it was selected. `QuestionAttempt` records each hint request, answer attempt, skip, or full-answer request. `LearnerQuestionState` stores per-question Leitner-style progress.
- `POST /api/chat/chat/` accepts free text or an `action` override. Supported actions are `answer`, `hint`, `skip`, `full_answer`, and `example_answer`.
- `full_answer` returns a stored `CourseQuestion.example_answer` first. If the field is blank, the app generates a full-mark answer with the model and caches it back onto the question with the `AI generated: ` prefix.
- Marking prompts must return JSON with `awarded_marks` and `explanation`. Invalid JSON falls back to zero marks with a generic retry message.
- Question selection sources currently include `explicit_question`, `default_unseen`, `default_due`, and `default_fallback`.
- Leitner score derivation and due-date scheduling live in `chat_api/progress.py`.
- The browser pages are intentionally thin wrappers over the JSON API. Keep the response shapes in `chat_api/views.py` aligned with the JavaScript in `core/templates/core/*.html` and the expectations in `core/tests.py`.

## Change Patterns

- For generic chat behavior shared across providers, edit `ai_chat/agents.py`.
- For OpenAI wiring or Chat Completions payload behavior, edit `ai_chat/openai_agent.py`.
- For persisted session storage, active-question state, attempts, or learner progression, edit `ai_chat/models.py` and/or `chat_api/services.py`.
- For course content, token-authenticated chat workflow, or response serialization, edit `chat_api/models.py`, `chat_api/services.py`, `chat_api/views.py`, `chat_api/auth.py`, `chat_api/rate_limits.py`, and `chat_api/urls.py` as needed.
- For next-question logic, selector overrides, or topic balancing, edit `chat_api/question_selector.py`.
- For progress, coverage, mastery, Leitner scheduling, or active-question summaries, edit `chat_api/progress.py`.
- For browser workflow changes, edit `core/views.py`, `core/urls.py`, and the relevant template in `core/templates/core/`.
- For env parsing or deployment behavior, edit `agentic_curiosity/settings.py`, `agentic_curiosity/env.py`, `Dockerfile`, `docker-entrypoint.sh`, `compose.yml`, `nginx.conf`, and `deploy/bootstrap.sh` as needed.
- For one provider only, add or modify a dedicated module in `ai_chat/`.
- Keep the OpenAI-shaped input contract centered on `Agent.create()`.
- Keep `Agent.ask()` as the convenience wrapper that returns plain text.
- Re-export public provider classes in `ai_chat/__init__.py`.
- Add tests in `ai_chat/tests.py`, `chat_api/tests.py`, or `core/tests.py` whenever behavior changes in those areas.
- For any new feature, add tests with the feature work and run both the focused new tests and the broader existing suite before considering the change complete.

## HTTP Routes

- `/`: Browser chat console.
- `/course-topics/`: Browser course studio.
- `/api/chat/login/`: Username/password login that returns a token and creates a Django session.
- `/api/chat/token/`: Issue or fetch the token for the current authenticated Django session.
- `/api/chat/courses/`: List courses on `GET`; create a course with nested topics/question types/questions on `POST`.
- `/api/chat/courses/<course_id>/questions/import/`: Import additional questions into an existing course.
- `/api/chat/sessions/`: Create a new tutoring session for a course, with optional selector overrides.
- `/api/chat/sessions/<session_id>/`: Fetch session state, active question, and progress summaries.
- `/api/chat/chat/`: Submit a learner message or `action` override for hint, mark, skip, or full-answer handling.
